# Phase 1: LLM-based Safe Word Filter

**Parent Plan:** [plan.md](./plan.md)
**Dependencies:** Phase 3 (optional — frequency data improves confidence)
**Date:** 2026-03-09
**Priority:** High
**Status:** Pending

## Overview

Use Gemini Flash API to batch-validate ~120K auto-generated compound words. 3-tier classification (VALID/INVALID/UNCERTAIN). Only remove words marked INVALID with high LLM confidence AND no corpus backing. Resumable with checkpoint files.

## Key Insights

- ~120K words generated by `mega-merge-dictionary.js` Phase 3 strategies (prefix x suffix combos)
- Gemini Flash: $0.10/1M input tokens, free tier = 15 RPM, 1M tokens/min
- At 100 words/batch (~1500 tokens input): ~1200 batches, ~80 min at free tier rate
- UNCERTAIN classification = KEEP (safe default)
- Frequency corpus (Phase 3) provides second signal — words appearing in Wikipedia are likely valid

## Requirements

- `GEMINI_API_KEY` env var (or `.env` file in backend root)
- Node.js 18+ (native fetch)
- Checkpoint file for resume capability
- Detailed output report with stats + removed words list

## Implementation Steps

### 1. Script setup (`backend/scripts/validate-dictionary-llm.js`)

1. Parse CLI args: `--checkpoint <path>`, `--batch-size <n>`, `--dry-run`, `--freq-file <path>`
2. Load env: `GEMINI_API_KEY` from `.env` or env var
3. Load dictionary from `vietnamese-wordlist.txt`
4. Load source words (all files in `sources/`) into a `sourceWords` Set — these are ground truth, skip validation
5. Compute `generatedWords` = dictionary minus sourceWords (these are the ~120K to validate)
6. Optionally load `wikipedia-frequency.json` (Phase 3 output) if available

### 2. Identify generated words

1. Re-run the source loading logic from `mega-merge-dictionary.js` (Phases 1-2 only, no compound gen)
2. Collect all source-backed words into `sourceWords` Set
3. `generatedWords` = full dictionary - sourceWords
4. Log: "Found X generated words to validate out of Y total"

### 3. Checkpoint system

1. Checkpoint file: `backend/scripts/.validate-checkpoint.json`
2. Structure:
   ```json
   {
     "version": 1,
     "batchSize": 100,
     "completedBatches": 847,
     "results": {
       "valid": ["word1", "word2"],
       "invalid": ["word3", "word4"],
       "uncertain": ["word5"],
       "errors": ["word6"]
     },
     "startedAt": "ISO",
     "lastUpdatedAt": "ISO"
   }
   ```
3. Save after every batch completes
4. On resume: skip completed batches, merge results

### 4. Gemini Flash API integration

1. Endpoint: `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent`
2. Prompt template per batch:
   ```
   You are a Vietnamese linguistics expert. Classify each word as VALID, INVALID, or UNCERTAIN.

   VALID = real Vietnamese word or compound (exists in dictionaries or common usage)
   INVALID = clearly not a real Vietnamese word (nonsensical combination)
   UNCERTAIN = might be valid but you're not sure

   When in doubt, mark as UNCERTAIN (we prefer to keep words rather than delete them).

   Words to classify:
   1. bất mát
   2. vô hẹp
   ...

   Respond in JSON format only:
   {"results": [{"word": "bất mát", "class": "INVALID"}, ...]}
   ```
3. Rate limiting: 15 RPM for free tier → 4-second delay between requests
4. Retry with exponential backoff on 429/500 errors (max 3 retries)
5. Parse JSON response, handle malformed responses (mark batch as error, retry once)

### 5. Cross-reference with frequency data

1. If `wikipedia-frequency.json` exists, load it
2. For each INVALID word: check if frequency > 0
3. If INVALID but freq > 2 → override to UNCERTAIN (keep)
4. Log overrides in report

### 6. Output generation

1. `backend/scripts/validation-report.json`:
   ```json
   {
     "totalWords": 255000,
     "sourceWords": 135000,
     "generatedWords": 120000,
     "validated": { "valid": 85000, "invalid": 28000, "uncertain": 7000 },
     "frequencyOverrides": 1200,
     "finalRemoved": 26800,
     "removedWords": ["word1", "word2", ...],
     "timestamp": "ISO"
   }
   ```
2. `backend/scripts/cleaned-wordlist.txt` — full dictionary minus confirmed invalids
3. Console summary with key stats

### 7. Safety guardrails

1. NEVER auto-delete — output to separate file, require manual review before replacing
2. If >50% of generated words flagged INVALID, abort with warning (likely prompt issue)
3. All source-backed words are exempt from removal
4. Log every removed word with LLM reasoning for audit

## Success Criteria

- Script completes validation of all ~120K generated words
- Checkpoint system allows resume after interruption
- <30% of generated words flagged (expected: 15-25%)
- Zero source-backed words removed
- Report provides clear audit trail of every removal
- Cost stays under $0.20

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| API key quota exceeded | Medium | Low | Free tier sufficient; checkpoint resume |
| LLM hallucination (flags valid words) | Medium | High | UNCERTAIN=keep, frequency cross-ref, manual review |
| Prompt injection from word content | Low | Low | Words are Vietnamese text, not commands |
| Rate limiting (429 errors) | High | Low | Exponential backoff + 4s delay |

## Security Considerations

- `GEMINI_API_KEY` in `.env` only, never committed (already in `.gitignore`)
- No user data sent to API — only dictionary words
- Checkpoint file contains no secrets
- Rate limit compliance: respect 15 RPM free tier
